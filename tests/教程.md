# ToolContext 教程

`ToolContext` 用于声明：某个工具需要、但模型不得自行生成的宿主侧值。它仅是一种注解语法；该参数不会出现在面向模型的输入 Schema、参数元数据列表或公开工具事件输入中。

## 必选 Context

当某个值必须由宿主在一次工具调用中提供时，使用 `ToolContext[T]`。Payload 可以是普通类，也可以是 dataclass。

```python
from dataclasses import dataclass

from easyharness import Agent, ModelConfig, ToolContext, tool


@dataclass(frozen=True)
class RequestContext:
    request_id: str
    user_id: str


@tool(
    name="read_profile",
    purpose="读取当前用户的资料。",
    when_to_use="当用户询问自己的资料时使用。",
    parameters=dict(),
    returns="用户资料摘要。",
    common_failures="当前请求不可用。",
)
def read_profile(request: ToolContext[RequestContext]) -> str:
    return f"用户 {request.user_id} 的资料"


agent = Agent(
    model=ModelConfig(model="example", api_key="example"),
    system_prompt="回答与用户资料有关的问题。",
    tools=[read_profile],
)
result = agent.run(
    "展示我的资料。",
    request=RequestContext(request_id="req-42", user_id="user-7"),
)
```

模型无法构造或提供 `request`；宿主会在当前这一次 `run()` 或 `stream()` 调用中，按参数名提供它。

## 可选 Context

当省略宿主值是合法的，并且省略时应解析为 `None`，请使用 `OptionalToolContext[T]`。

```python
from easyharness import OptionalToolContext, tool


@tool(
    name="read_profile_summary",
    purpose="在请求数据可用时读取用户资料。",
    when_to_use="当请求可能来自匿名用户时使用。",
    parameters=dict(),
    returns="用户资料摘要或匿名标记。",
    common_failures="资料查询失败。",
)
def read_profile_summary(
    request: OptionalToolContext[RequestContext],
) -> str:
    if request is None:
        return "匿名请求"
    return f"用户 {request.user_id} 的资料"


assert read_profile_summary() == "匿名请求"
```

当可选参数没有源码默认值时，被装饰的可调用对象会在可检查的签名中暴露 `request=None`。因此，调用方可以在普通参数之后省略可选 Context：

```python
def render_message(
    text: str,
    request: OptionalToolContext[RequestContext],
) -> str:
    return text if request is None else f"{text}:{request.request_id}"


assert render_message("你好") == "你好"
```

## 默认值解析顺序

直接调用和 Agent 工具调用会对每个 Context 参数使用相同的解析顺序：

1. 使用按 Context 参数名显式提供的宿主值。
2. 否则，使用原始函数参数上声明的默认值。
3. 否则，对 `OptionalToolContext[T]` 注入 `None`。
4. 否则，在执行工具函数体前安全失败。

无论 Context 是必选还是可选，源码中声明的默认值都会被保留：

```python
DEFAULT_REQUEST = RequestContext(request_id="default", user_id="system")


@tool(
    name="read_default_profile",
    purpose="需要时使用宿主默认值读取用户资料。",
    when_to_use="仅当允许使用默认资料时使用。",
    parameters=dict(),
    returns="用户资料摘要。",
    common_failures="请求 Context 无效。",
)
def read_default_profile(
    request: ToolContext[RequestContext] = DEFAULT_REQUEST,
) -> str:
    return request.user_id


assert read_default_profile() == "system"
```

## 支持的 Payload 类型

Context Payload 必须是一个具体且可在运行时检查的类型表达式。以下类型支持递归组合：

- 普通类，包括已声明父类的子类。
- `object`。
- `dict[K, V]`、`list[T]`、`set[T]`、`tuple[T, ...]`，以及 `tuple[str, int]` 这类固定长度元组。

```python
@tool(
    name="inspect_context_payload",
    purpose="检查宿主拥有的结构化数据。",
    when_to_use="仅在宿主提供已验证的结构化数据时使用。",
    parameters=dict(),
    returns="固定状态标记。",
    common_failures="Context Payload 无效。",
)
def inspect_context_payload(
    records: ToolContext[dict[str, list[RequestContext]]],
    marker: ToolContext[tuple[str, int]],
    labels: OptionalToolContext[set[str]],
) -> str:
    del records, marker, labels
    return "已接受"
```

系统不会执行类型强制转换。例如，字典不能替代声明为 `RequestContext` 的对象。

## 声明限制

不要使用 Payload 联合类型。它们会在 `@tool` 注册函数时失败，包括嵌套在受支持容器中的联合类型：

```python
request: ToolContext[RequestContext | AdminContext]
request: ToolContext[dict[str, RequestContext | AdminContext]]
request: ToolContext[RequestContext] | AdminContext
```

不要在工具 Context 合约中使用 `Any`、`None`、`Protocol`、不受支持的 `typing` 形式、仅位置参数、`*args` 或 `**kwargs`。Context 参数必须位于所有普通的位置或关键字参数之后。

`ToolContext` 和 `OptionalToolContext` 不是基类，不能被继承。它们是注解辅助语法，而不是领域模型的继承点。

## 兼容性与迁移

`ToolContext[T] | None` 为兼容性而保留，但会发出 `DeprecationWarning`。新代码应使用 `OptionalToolContext[T]`。

`ToolContext[object]` 为兼容性而接受显式传入的 `None`，但在使用该值时会发出 `RuntimeWarning`，建议迁移为 `OptionalToolContext[object]`。只要 `None` 是预期合约的一部分，就应优先使用可选注解。

## 共享 Agent 合约

多个工具可以使用相同的 Context 参数名，前提是它们的标准化 Payload 注解完全相同。必选和可选声明可以针对同一个共享名称混用。若同名参数的 Payload 注解不同，会在创建 Agent 时被拒绝。

## 失败、作用域与隐私

缺少必选 Context 或 Payload 类型错误时，工具函数体不会执行。错误信息会标识参数和预期类型，但不会渲染传入的 Context 值。未知的 Context 参数名会在 Agent 调用开始前被拒绝。

Context 值只在一次 `run()` 或 `stream()` 调用期间存活。EasyHarness 不会构造、序列化、缓存或复用它们。工具输出仍是普通的工具输出：如果工具主动返回了 Context 数据，返回的数据可能会对模型可见。

## 检查清单

1. 对模型不可见的宿主拥有值使用 `ToolContext[T]`。
2. 当省略值应解析为 `None` 时使用 `OptionalToolContext[T]`。
3. 不要将 Context 参数名写入 `parameters` 元数据。
4. 将 Context 参数放在普通位置或关键字参数之后。
5. 使用一个具体、受支持且不含联合类型的 Payload 类型。
6. 当工具需要 Context 时，按 Context 参数的确切名称传入宿主值。
7. 除非工具显式脱敏，否则应将工具输出视为对模型可见。
